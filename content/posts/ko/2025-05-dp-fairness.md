# 차등 프라이버시의 공정성 문제

## DP가 오히려 불평등을 심화시킨다?

차등 프라이버시는 개인을 보호하지만, 의도치 않게 **특정 집단에 더 불리한 결과**를 만들 수 있습니다.

### 구체적 시나리오

5개 소득 그룹 데이터를 DP로 분석한다고 가정합니다:
- 다수 그룹 4개: 각 1,000명
- 소수 그룹 1개: 50명 (희귀 질환자, 소수 민족 등)

DP는 **단일 임계값(τ)**으로 모든 데이터를 클리핑합니다. 다수 그룹에 최적화된 τ가 소수 그룹과 맞지 않으면:

```
다수 그룹: τ=86,998 → 80%가 포용됨 → 오차 0.8%
소수 그룹: τ=86,998 → 단 6%만 포용됨 → 오차 46.5%!
```

소수 그룹의 평균 소득이 175,821원인데, 86,998원으로 잘려서 거의 모든 데이터가 왜곡됩니다.

## TaupT의 해법: 하이브리드 공정성 보정

TaupT는 단순히 "모두에게 같은 τ"를 강요하지 않습니다.

```
Step 1: 전체 데이터로 최적 τ* 결정

Step 2: 각 그룹의 coverage 확인
        coverage = "τ* 이하인 데이터 비율"

Step 3: 그룹 분류
        coverage ≥ 80% → 잘 포용됨 → τ* 그대로 사용
        coverage < 80% → 잘 포용되지 않음 → 해당 그룹만 독립 처리

Step 4: 불충분 그룹은 자기 데이터로 최적 τ_g 독립 선택
```

핵심은 **병렬 합성 정리**: 그룹이 서로 분리되어 있으므로, 각 그룹이 ε=1.0으로 독립 실행해도 전체는 여전히 ε=1.0입니다. 추가 프라이버시 비용 없음.

## 실험 결과

| 방법 | 소수 그룹(N=50) 오차 | 전체 오차 |
|---|---|---|
| 기존 방식 (P90 고정) | 62.26% | 7.75% |
| 단순 DP | 46.78% | 0.80% |
| **TaupT 하이브리드** | **5.53%** | **0.80%** |

- 소수 그룹 오차: 46.78% → **5.53% (8.5배 개선)**
- 다수 그룹: 변화 없음 (0.80% 유지)

**"문제가 없는 그룹은 건드리지 않고, 문제가 있는 그룹만 선택적으로 개입하여 극적 개선"**

## 왜 이게 중요한가

의료 AI에서 소수 집단에 부정확한 진단이 나온다면, DP를 "공정하게 적용했다"는 말이 무의미합니다.

금융 신용평가에서 특정 인구 집단의 점수가 DP로 인해 더 부정확해진다면, 이것은 기술적 차별입니다.

TaupT의 공정성 보정은 "DP를 모두에게 공평하게 적용하는 것"이 기술적으로 어떤 의미인지를 재정의합니다.
