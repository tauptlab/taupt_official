# 연합학습 + 차등 프라이버시

## 문제: 데이터는 있지만 공유할 수 없다

폐암 진단 AI를 만들고 싶습니다. 하지만 한 병원의 데이터만으로는 모델의 정확도가 부족합니다. 그렇다고 환자 데이터를 다른 병원에 보낼 수는 없습니다.

이것이 **데이터 사일로** 문제입니다. 금융, 의료, 공공 분야에서 가장 가치 있는 데이터가 바로 이런 상황에 묶여 있습니다.

## 해법: 데이터는 이동하지 않습니다

연합학습은 데이터를 공유하는 대신, **AI 모델만 오가며 학습**시킵니다.

```
  병원 A              병원 B              병원 C
  [환자 데이터]        [환자 데이터]        [환자 데이터]
       ↓                   ↓                   ↓
  로컬에서 학습         로컬에서 학습         로컬에서 학습
       ↓                   ↓                   ↓
  모델 업데이트         모델 업데이트         모델 업데이트
       └───────────────────┼───────────────────┘
                           ↓
                    중앙에서 안전하게 집계
                           ↓
                   더 정확한 공동 AI 모델
```

데이터는 한 바이트도 병원 밖으로 나가지 않습니다.

## 그래도 안전하지 않다?

연합학습만으로는 충분하지 않습니다. 모델 업데이트(그래디언트) 자체에서 원본 데이터를 **역추론**할 수 있다는 것이 밝혀졌습니다.

실제 연구에서 그래디언트로부터 원본 이미지를 복원하는 데 성공했습니다. **모델 업데이트도 보호해야 합니다.**

## TaupT: 그래디언트에도 DP를 적용

TaupT는 각 참여자의 모델 업데이트에 차등 프라이버시 노이즈를 추가합니다.

그러나 여기서 고차원 문제가 발생합니다. 딥러닝 모델의 그래디언트는 수십~수백 차원입니다. 기존 DP 방식으로는 노이즈가 폭증하여 학습이 불가능해집니다.

TaupT의 **AC-PQ(방향 보존형 클리핑)** 기술이 바로 이 문제를 해결합니다:

- 50차원 그래디언트에서 **코사인 유사도 0.879** 보존
- 기존 모든 방법은 0.1 이하로 방향이 파괴됨
- **TaupT만이 50차원 연합학습을 실용적으로 가능하게 함**

## 악의적 참여자 방어

연합학습의 또 다른 위협: 악의적 참여자가 의도적으로 잘못된 그래디언트를 보내 모델을 오염시키는 **Byzantine 공격**.

TaupT의 실험에서 5% Byzantine 공격자가 있는 50차원 환경:
- 기존 방법들: 코사인 유사도 음수 (완전히 방향 파괴)
- TaupT: 0.879 유지 (공격에도 안정적)

## 도입 형태

TaupT는 Flower(Flwr) 연합학습 프레임워크와 완전히 호환됩니다. 기존 연합학습 파이프라인에 TaupT를 플러그인으로 추가하면 즉시 DP 보호가 적용됩니다.

의료, 금융, 공공 분야에서 데이터를 공유하지 않고 협력 AI를 구축하고자 하는 조직에게 TaupT는 현재 유일한 실용적 솔루션입니다.
