# 고차원 데이터의 DP 문제

## 차원이 많아지면 왜 문제가 생기나

온도, 습도, 전압, 진동 등 20가지 센서를 동시에 수집하는 IoT 시스템을 생각해봅시다.

기존 차등 프라이버시 방식은 **각 열(차원)에 독립적으로 노이즈를 추가**합니다. 프라이버시 예산 ε=1.0이 있다면, 20개 열에 ε/20 = 0.05씩 나눠 써야 합니다.

차원이 많아질수록:
- 각 열에 배정되는 예산이 줄어듭니다
- 예산이 적으면 노이즈가 커집니다
- 분석 결과가 의미를 잃어버립니다

**20차원에서 기존 방식의 코사인 유사도: ~0.26 (원래 방향과 74% 다름)**

## TaupT의 핵심 혁신: L2-노름 기반 처리

TaupT의 특허 기술 **AC-PQ(Adaptive Clipping via Private Quantiles)**는 다른 접근법을 사용합니다.

```
기존 방식: 20차원 벡터 → 20개로 쪼개서 각각 처리
TaupT:    20차원 벡터 → L2-노름(스칼라 하나)으로 환원 → 1번 처리
```

핵심 수식:

```
클리핑: c'ᵢ = cᵢ × min(1, τ* / ‖cᵢ‖₂)
```

- 벡터의 **크기만** τ*로 줄입니다
- **방향은 정확히 보존**됩니다 (cos(cᵢ, c'ᵢ) = 1)
- 차원 수와 무관하게 동일한 ε 비용으로 처리합니다

## 실험 결과: 경쟁자가 없는 영역

### 20차원 IoT 센서 데이터

| 방법 | 전체 MAE | TaupT 대비 |
|---|---|---|
| **TaupT (AC-PQ)** | **51.94** | **기준** |
| Subsample | 82.27 | +58% 나쁨 |
| SmoothSens (Google 계열) | 88.10 | +70% 나쁨 |
| Trimmed | 123.26 | +137% 나쁨 |
| DPQ-2Stage | 683.29 | +1216% 나쁨 |

### 50차원 연합학습 그래디언트

10,000개 참여자가 50차원 그래디언트를 보내는 상황 (5% 악의적 공격 포함):

| 방법 | 코사인 유사도 | 상태 |
|---|---|---|
| **TaupT (AC-PQ)** | **0.879** | ✅ 작동 |
| SmoothSens | 0.036 | ❌ 실패 |
| Subsample | 0.007 | ❌ 실패 |
| DPQ-2Stage | -0.017 | ❌ 실패 |
| Trimmed | -0.005 | ❌ 실패 |

코사인 유사도 0은 "랜덤 방향", 음수는 "반대 방향"을 의미합니다. **50차원에서 TaupT만이 유일하게 작동합니다.**

## 포지셔닝

| 차원 수 | TaupT 순위 | 상황 |
|---|---|---|
| 1차원 | 5위/6위 | Google, Harvard 라이브러리로 충분 |
| 20차원 | **1위** (2위 대비 58% 우위) | IoT, CCTV 분석에 최적 |
| 50차원 | **유일한 작동 방법** | 연합학습 그래디언트 집계에 필수 |

TaupT의 핵심 가치는 **"고차원 벡터 DP의 유일한 솔루션"**입니다.
