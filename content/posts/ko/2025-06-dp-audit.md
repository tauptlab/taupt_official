# DP 감사: 올바르게 작동하는가

## "적용했다"와 "작동한다"는 다르다

회사가 "차등 프라이버시를 적용했습니다"라고 말할 때, 이것은 두 가지 의미일 수 있습니다:

1. DP 라이브러리를 코드에 import했다
2. DP가 실제로 올바르게 작동하며 개인정보를 보호하고 있다

안타깝게도 대부분의 경우 1번에 해당합니다. **구현 버그, 설정 오류, 예산 추적 오류** 등으로 실제로는 보호되지 않을 수 있습니다.

## 실제 발생하는 문제들

### 예산 누수

```python
# 잘못된 구현: 같은 쿼리를 반복해도 예산이 차감되지 않음
def query(data, epsilon):
    noise = laplace(sensitivity / epsilon)
    return mean(data) + noise  # 예산 추적 없음!
```

위 코드는 ε을 매번 쓰면서도 추적하지 않습니다. 1000번 쿼리해도 예산이 소진되지 않습니다.

### 교묘한 쿼리 조합 공격

두 개의 DP 쿼리를 조합하면 노이즈가 상쇄될 수 있습니다:

```
쿼리 1: "직원 전체 평균 소득" = 실제값 + 노이즈 A
쿼리 2: "직원 A 제외 평균 소득" = 실제값' + 노이즈 B
쿼리 1 - 쿼리 2 ≈ 직원 A의 소득 (노이즈 대부분 상쇄)
```

## TaupT 3중 감사 시스템

TaupT는 세 가지 독립적인 방법으로 DP 작동을 자동 검증합니다.

### 1. 공격 시뮬레이션

```
자기-상쇄 쿼리 쌍을 자동 생성
→ 노이즈 상쇄 시도
→ 예산 초과 여부 감지
→ 비정상 패턴 차단
```

### 2. 수학적 일관성 검증

프라이버시 손실이 이론대로 누적되는지 **고정밀 소수점 추적**으로 검증합니다.

```go
// TaupT의 고정밀 예산 추적
type DecimalEpsilon struct {
    value *big.Float  // 부동소수점 오차 없는 정밀 계산
}
```

### 3. 슬라이딩 윈도우 감사

실시간으로 쿼리 패턴을 분석하여 비정상적인 정보 추출 시도를 탐지합니다.

## 규제 준수 보고서

감사 결과는 자동으로 **컴플라이언스 보고서**로 생성됩니다.

- ε 소비 이력 및 잔여 예산
- 감사 통과/실패 기록
- 이상 탐지 로그

감사 기관이나 규제 당국에 "우리의 DP가 올바르게 작동하고 있다"는 증빙으로 제출할 수 있습니다. 개인정보보호법, GDPR 준수 증빙에 활용 가능합니다.

## 결론

DP는 강력한 기술이지만, 올바르게 구현되고 지속적으로 모니터링되어야만 실제 보호 효과를 냅니다. TaupT의 3중 감사 시스템은 "적용했다"는 주장을 "작동한다"는 증명으로 바꿉니다.
